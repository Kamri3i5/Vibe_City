# 🌆 VibeCity — Живой Пульс Ташкента

VibeCity — интерактивная «живая» карта города, которая помогает жителям и гостям Ташкента находить самые активные локации или, наоборот, спокойные места для отдыха в реальном времени.

**Демо:** https://kamri3i5.github.io/Vibe_City/

## ✨ Основные фишки
- **Живая карта:** визуализация активности города на базе Leaflet + кластеризация маркеров.
- **Живые события с Afisha.uz:** концерты, театр, выставки и городские новости — реальные данные, обновляются автоматически каждые 6 часов.
- **Вайб-чеки:** отметки 🔥 Кайф / 💀 Глухо / 😭 Скучно — делись атмосферой места.
- **Пульс города:** события, голоса, общий вайб и погода (Open-Meteo) — только реальные данные.
- **Топ сегодня и «Сейчас горит»:** рейтинги самых горячих мест.
- **Лента активности:** реальные записи — твои вайб-чеки и свежие события города.
- **Такси в один тап:** кнопка «Такси сюда» открывает маршрут в Yandex Go.
- **Карта метро:** интерактивная схема трёх линий с переходом к станции на карте.
- **Профиль:** уровни и ранги за реальные действия, история вайб-чеков, вход через Google.
- **3 языка:** русский, английский, узбекский. Тёмная и светлая темы.
- **Адаптивность:** нижняя навигация и bottom-sheet-панели на мобильных.

## 🔄 Пайплайн живых данных
```
Afisha.uz RSS ──▶ scripts/fetch-afisha.mjs ──▶ data/afisha.json ──▶ фронтенд
                  (GitHub Action, каждые 6 ч)      (в репозитории)
```
- Парсер: `node scripts/fetch-afisha.mjs` (Node 18+, без зависимостей).
- Расписание: `.github/workflows/update-afisha.yml` (cron + ручной запуск в Actions).
- Фронт кэширует последний ответ в localStorage — работает и офлайн.
- События с распознанной площадкой (Humo Arena, Театр Навои и др.) появляются маркерами на карте.

## 📁 Структура проекта
```
index.html                        — основное приложение (все экраны и оверлеи)
style.css                         — дизайн-система (CSS-переменные) + компоненты
script.js                         — логика: state, storage, i18n, карта, afisha, auth
data/afisha.json                  — живые события (генерируется автоматически)
scripts/fetch-afisha.mjs          — парсер RSS Afisha.uz
.github/workflows/update-afisha.yml — автообновление событий по расписанию
admin.html / admin.css / admin.js — админ-панель (места, события, телеграм-парсер)
```

## 🛠 Технологии
- **Frontend:** HTML5, Vanilla CSS, JavaScript (без сборки — деплой на GitHub Pages как есть).
- **Карты:** [Leaflet.js](https://leafletjs.com/) + CartoDB (Dark Matter / Voyager).
- **Иконки:** [Lucide Icons](https://lucide.dev/) (версия зафиксирована).
- **Погода:** [Open-Meteo](https://open-meteo.com/) (без ключа).
- **Шрифты:** Inter (Google Fonts).

## 🚀 Как запустить локально
1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/Kamri3i5/Vibe_City.git
   ```
2. Запустите любой статический сервер в папке проекта, например:
   ```bash
   python -m http.server 8080
   ```
   и откройте http://localhost:8080. (Открытие `index.html` напрямую тоже работает, но Google Sign-In требует http/https origin.)

## 🔐 Админ-панель
`admin.html` — внутренний инструмент команды. Роли задаются в `ADMIN_PINS` в `admin.js`:
- **owner** (PIN 1234) — полный доступ: места, ивенты, удаление, сброс данных;
- **editor** (PIN 7777) — только добавление/редактирование мест и запуск ивентов.

⚠️ Это **клиентская** защита для демо: данные живут в `localStorage` браузера и изменения видны только локально. Для продакшна нужен бэкенд (см. PLAN.md).
